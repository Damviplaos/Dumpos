# เอกสารความต้องการ

## 1. ภาพรวมแอปพลิเคชัน

### 1.1 ชื่อแอปพลิเคชัน
ระบบขายหน้าร้าน POS (Point of Sale) แบบ Multi-tenant SaaS

### 1.2 คำอธิบาย
ระบบ POS แบบมืออาชีพสำหรับการขายสินค้าหน้าร้าน รองรับหลายร้านค้า (Multi-tenant) บนเซิร์ฟเวอร์เดียว ครอบคลุมการขาย การจัดการสินค้า การจัดการสต็อก รายงาน การตั้งค่าร้าน ระบบจัดการยศและสิทธิ์แบบละเอียด ระบบตรวจจับป้องกันการโกง ระบบป้องกันการยักยอกทรัพย์ ระบบใบเตือนพนักงาน ระบบ Super Admin และการแจ้งเตือนผ่าน Telegram Bot รองรับ 4 ภาษา (ไทย ลาว อังกฤษ จีน) ออกแบบให้ใช้งานง่าย สวยงาม ทันสมัย

## 2. ผู้ใช้และสถานการณ์การใช้งาน

### 2.1 ผู้ใช้เป้าหมาย
- Super Admin (เจ้าของโปรแกรม/เว็บไซต์): จัดการระบบทั้งหมดข้ามร้าน สร้าง/เปิด/ปิดร้านค้า สร้าง account เจ้าของร้าน ดูข้อมูลทุกร้าน จัดการ Telegram Token ของทุกร้าน
- เจ้าของร้าน (Store Owner): จัดการระบบทั้งหมดของร้านตนเอง เพิ่ม/จัดการพนักงาน ดูรายงาน ตั้งค่าร้าน จัดการยศและสิทธิ์ ตรวจสอบการโกง ออกใบเตือนพนักงาน ตั้งค่า Telegram Bot
- พนักงานขาย (Cashier): ขายสินค้า รับชำระเงิน
- พนักงานตามยศที่กำหนด: ทำงานตามสิทธิ์ที่ได้รับจากยศ

### 2.2 สถานการณ์การใช้งานหลัก
- Super Admin เข้าหน้า Super Admin Panel เพื่อสร้างร้านค้าใหม่ สร้าง account store_owner เปิด/ปิดร้าน ดูข้อมูลทุกร้าน ดู Audit Log และ Fraud Alert ของทุกร้าน ตั้งค่าระบบ จัดการ Telegram Token
- เจ้าของร้านเพิ่ม/จัดการพนักงานในร้านตัวเอง ตั้งค่าร้าน ดูรายงานร้านตัวเอง
- พนักงานขายใช้ระบบขายสินค้าหน้าร้าน รับชำระเงินจากลูกค้า
- เจ้าของร้านจัดการสินค้า อัปโหลดรูปภาพสินค้า ติดตามสต็อก ดูรายงานยอดขายและกำไร
- เจ้าของร้านสร้างและจัดการยศพร้อมกำหนดสิทธิ์ละเอียด
- เจ้าของร้านตรวจสอบ Audit Log และพฤติกรรมน่าสงสัยของพนักงาน
- เจ้าของร้านออกใบเตือนพนักงาน (ใบเหลือง/ใบแดง) และดูประวัติการเตือน
- เจ้าของร้านตั้งค่า Telegram Bot Token และ Chat ID เพื่อรับแจ้งเตือนอัตโนมัติ
- เจ้าของร้านสอบถามข้อมูลยอดขายผ่าน Telegram Bot
- ผู้ใช้สลับภาษาระหว่าง ไทย ลาว อังกฤษ จีน ผ่านปุ่มธงชาติที่ header
- ระบบป้องกันการแก้ไข/ลบบิลที่ชำระเงินแล้ว และแจ้งเตือนการพยายามยักยอกทรัพย์ผ่าน Telegram

## 3. โครงสร้างหน้าและฟังก์ชัน

### 3.1 โครงสร้างหน้าหลัก

```
├── หน้าเข้าสู่ระบบ (รองรับสลับภาษา)
├── Super Admin Panel (Super Admin เท่านั้น)
│   ├── จัดการร้านค้าทั้งหมด
│   ├── สร้างร้านใหม่ + สร้าง account store_owner
│   ├── เปิด/ปิดร้าน
│   ├── ดูสถิติแต่ละร้าน
│   ├── จัดการ Users ข้ามร้าน
│   ├── ดู Audit Log ทุกร้าน
│   ├── ดู Fraud Alert ทุกร้าน
│   ├── ตั้งค่าระบบ (System Settings)
│   ├── ดูรายงานรวมทุกร้าน
│   └── จัดการ Telegram Token ของทุกร้าน
├── Dashboard (เจ้าของร้าน)
├── หน้าขายสินค้า (Cashier/Sales Screen)
├── การจัดการสินค้า (Product Management)
│   ├── รายการสินค้า
│   └── เพิ่ม/แก้ไข/ลบสินค้า
├── การจัดการสต็อก (Inventory)
├── รายงาน (Reports)
│   ├── รายงานยอดขาย
│   ├── รายงานสินค้าขายดี
│   └── รายงานกำไร
├── ประวัติการขาย (Transaction History)
├── การจัดการผู้ใช้
│   ├── รายการผู้ใช้ (แสดง badge สถานะใบเตือน)
│   ├── ออกใบเตือนพนักงาน
│   └── ดูประวัติการเตือน
├── การจัดการยศและสิทธิ์ (Role Management)
│   ├── รายการยศ
│   ├── สร้าง/แก้ไข/ลบยศ
│   └── กำหนดสิทธิ์ยศ
├── ระบบตรวจจับการโกง (Fraud Monitor)
│   ├── Audit Log
│   ├── Alert การโกง
│   └── สถิติพฤติกรรมน่าสงสัย
└── การตั้งค่าร้าน
    ├── ข้อมูลร้าน
    ├── การตั้งค่าทั่วไป
    └── Telegram Bot Integration
```

### 3.2 หน้าเข้าสู่ระบบ
- แสดงปุ่มธงชาติ 4 ภาษา (🇹🇭 ไทย 🇱🇦 ลาว 🇬🇧 อังกฤษ 🇨🇳 จีน) ให้ผู้ใช้เลือกภาษาก่อน login
- ผู้ใช้กรอกชื่อผู้ใช้และรหัสผ่านเพื่อเข้าสู่ระบบ
- ระบบตรวจสอบข้อมูลและนำผู้ใช้ไปยังหน้าที่เหมาะสม:
  - Super Admin → Super Admin Panel
  - Store Owner → Dashboard
  - Cashier → หน้าขายสินค้า
- ระบบตรวจสอบสถานะใบเตือน: หากพนักงานมีสถานะ red_card ระบบ block การ login และแสดงข้อความ (แปลตามภาษาที่เลือก)
- ระบบบันทึก login action ใน Audit Log

### 3.3 Super Admin Panel (Super Admin เท่านั้น)

#### 3.3.1 จัดการร้านค้าทั้งหมด
- แสดงรายการร้านค้าทั้งหมด พร้อมชื่อร้าน เจ้าของร้าน (store_owner) สถานะ (active/inactive)
- ผู้ใช้ดูรายละเอียดร้าน
- ผู้ใช้ดูสถิติแต่ละร้าน: ยอดขาย จำนวนพนักงาน

#### 3.3.2 สร้างร้านใหม่ + สร้าง account store_owner
- ผู้ใช้กรอกชื่อร้าน ที่อยู่ เบอร์โทรศัพท์
- ผู้ใช้กรอกข้อมูล store_owner: ชื่อผู้ใช้ รหัสผ่าน อีเมล
- ระบบสร้างร้านใหม่ (store_id)
- ระบบสร้าง account store_owner พร้อมกำหนด role = store_owner และ store_id
- ระบบบันทึก action ใน Audit Log

#### 3.3.3 เปิด/ปิดร้าน
- ผู้ใช้เลือกร้านที่ต้องการเปิด/ปิด
- ระบบเปลี่ยนสถานะร้าน (active/inactive)
- หากปิดร้าน ระบบ block การ login ของ store_owner และพนักงานในร้านนั้น
- ระบบบันทึก action ใน Audit Log

#### 3.3.4 จัดการ Users ข้ามร้าน
- แสดงรายการผู้ใช้ทั้งหมดจากทุกร้าน พร้อมชื่อ ยศ ร้านที่สังกัด สถานะใบเตือน
- ผู้ใช้ค้นหาและกรองผู้ใช้ตามร้าน ยศ สถานะ
- ผู้ใช้แก้ไขข้อมูลผู้ใช้ข้ามร้านได้

#### 3.3.5 ดู Audit Log ทุกร้าน
- แสดงรายการ action ทั้งหมดจากทุกร้าน พร้อมวันที่ เวลา ผู้ใช้ ร้าน action รายละเอียด
- ผู้ใช้กรองตามร้าน user action วันที่

#### 3.3.6 ดู Fraud Alert ทุกร้าน
- แสดงรายการ alert ทั้งหมดจากทุกร้าน พร้อมระดับความเสี่ยง ร้าน ผู้ใช้ ประเภทการโกง
- ผู้ใช้กรองตามร้าน ระดับความเสี่ยง วันที่

#### 3.3.7 ตั้งค่าระบบ (System Settings)
- ผู้ใช้ตั้งค่าระบบในหน้าเดียว ไม่ต้องแก้ code
- ตั้งค่าพารามิเตอร์ต่างๆ ของระบบ

#### 3.3.8 ดูรายงานรวมทุกร้าน
- แสดงสรุปยอดขายรวมทุกร้าน
- แสดงกำไรรวมทุกร้าน
- แสดงจำนวนออเดอร์รวมทุกร้าน
- แสดงกราฟยอดขายรวม

#### 3.3.9 จัดการ Telegram Token ของทุกร้าน
- แสดงรายการร้านพร้อม Telegram Bot Token และ Chat ID ที่ store_owner แต่ละร้านตั้งค่าไว้
- ผู้ใช้ดู แก้ไข หรือลบ Telegram Token ของร้านใดก็ได้

### 3.4 Dashboard (เจ้าของร้าน)
- แสดงสรุปยอดขายวันนี้ของร้านตัวเอง
- แสดงกำไรวันนี้ของร้านตัวเอง
- แสดงจำนวนออเดอร์วันนี้ของร้านตัวเอง
- แสดงกราฟยอดขายรายวัน/รายสัปดาห์/รายเดือนของร้านตัวเอง

### 3.5 หน้าขายสินค้า (Cashier/Sales Screen)

#### 3.5.1 การเลือกและค้นหาสินค้า
- แสดงรายการสินค้าของร้านตัวเองแบ่งตามหมวดหมู่ พร้อมรูปภาพสินค้า (แสดงจาก Supabase Storage public URL)
- ผู้ใช้คลิกเลือกสินค้าเพื่อเพิ่มเข้าตะกร้า
- ผู้ใช้ค้นหาสินค้าด้วยชื่อหรือรหัสสินค้า
- ผู้ใช้สแกนบาร์โค้ดเพื่อเพิ่มสินค้าเข้าตะกร้า

#### 3.5.2 ตะกร้าสินค้า
- แสดงรายการสินค้าที่เลือก พร้อมชื่อ ราคา จำนวน
- ผู้ใช้แก้ไขจำนวนสินค้าในตะกร้า
- ผู้ใช้ลบสินค้าออกจากตะกร้า
- แสดงยอดรวมทั้งหมด
- ราคาสินค้ามาจากฐานข้อมูลเท่านั้น ไม่สามารถแก้ไขราคาในตะกร้าได้

#### 3.5.3 การชำระเงิน
- ผู้ใช้เลือกวิธีชำระเงิน: เงินสด, บัตรเครดิต/เดบิต, QR Code
- กรณีเงินสด: ผู้ใช้กรอกจำนวนเงินที่รับ ระบบคำนวณเงินทอน
- กรณีบัตรเครดิต/เดบิต: ระบบบันทึกการชำระด้วยบัตร
- กรณี QR Code: ระบบแสดง QR Code สำหรับชำระเงิน
- ระบบบันทึกรายการขายและอัปเดตสต็อกสินค้า
- ระบบเปลี่ยนสถานะบิลเป็น completed
- ระบบบันทึก sales action ใน Audit Log
- ระบบแสดงหน้าจอยืนยันการขายสำเร็จ
- ผู้ใช้พิมพ์ใบเสร็จ (ถ้าต้องการ)

### 3.6 การจัดการสินค้า (Product Management)

#### 3.6.1 รายการสินค้า
- แสดงรายการสินค้าของร้านตัวเองทั้งหมด พร้อมรูปภาพ (thumbnail จาก Supabase Storage public URL) ชื่อ หมวดหมู่ ราคา สต็อก
- ผู้ใช้ค้นหาสินค้า
- ผู้ใช้กรองสินค้าตามหมวดหมู่

#### 3.6.2 เพิ่มสินค้า
- ผู้ใช้กรอกข้อมูลสินค้า: ชื่อ, รหัสสินค้า, บาร์โค้ด, หมวดหมู่, ราคา, ต้นทุน, สต็อก
- ผู้ใช้อัปโหลดรูปภาพสินค้า (ไม่บังคับ):
  - เลือกรูปจาก Gallery
  - ถ่ายรูปจากกล้องโดยตรง
- ระบบอัปโหลดรูปภาพไปยัง Supabase Storage bucket product-images
- ระบบจัดเก็บรูปภาพในโฟลเดอร์ products/{product_id}/{filename}
- ระบบบันทึก image_url เป็น public URL จาก Supabase Storage ใน products table
- ระบบบันทึกสินค้าใหม่พร้อม store_id
- ระบบบันทึก add product action ใน Audit Log

#### 3.6.3 แก้ไขสินค้า
- ผู้ใช้เลือกสินค้าของร้านตัวเองที่ต้องการแก้ไข
- ผู้ใช้แก้ไขข้อมูลสินค้า
- ผู้ใช้เปลี่ยนรูปภาพสินค้า หรืออัปโหลดรูปใหม่
- ระบบบันทึกการเปลี่ยนแปลง
- ระบบบันทึก edit product action ใน Audit Log

#### 3.6.4 ลบสินค้า
- ผู้ใช้เลือกสินค้าของร้านตัวเองที่ต้องการลบ
- ระบบยืนยันการลบ
- ระบบลบสินค้าออกจากระบบ
- ระบบบันทึก delete product action ใน Audit Log

### 3.7 การจัดการสต็อก (Inventory)
- แสดงรายการสินค้าของร้านตัวเองพร้อมจำนวนสต็อกปัจจุบัน
- แสดงสินค้าที่สต็อกใกล้หมด (ต่ำกว่าจำนวนที่กำหนด)
- ผู้ใช้แก้ไขจำนวนสต็อก
- ผู้ใช้ระบุเหตุผลการแก้ไขสต็อก
- ระบบบันทึกการแก้ไขสต็อกพร้อมเหตุผลใน Audit Log
- ระบบแจ้งเตือนเมื่อสินค้าใกล้หมด

### 3.8 รายงาน (Reports)

#### 3.8.1 รายงานยอดขาย
- ผู้ใช้เลือกช่วงเวลา: รายวัน, รายสัปดาห์, รายเดือน
- แสดงยอดขายรวมของร้านตัวเอง จำนวนออเดอร์ ยอดขายเฉลี่ย
- แสดงกราฟยอดขาย

#### 3.8.2 รายงานสินค้าขายดี
- ผู้ใช้เลือกช่วงเวลา
- แสดงรายการสินค้าขายดี 10 อันดับแรกของร้านตัวเอง พร้อมจำนวนที่ขาย

#### 3.8.3 รายงานกำไร
- ผู้ใช้เลือกช่วงเวลา
- แสดงยอดขาย ต้นทุน กำไร อัตรากำไรของร้านตัวเอง

### 3.9 ประวัติการขาย (Transaction History)
- แสดงรายการขายย้อนหลังของร้านตัวเอง พร้อมวันที่ เวลา ยอดรวม วิธีชำระเงิน สถานะ (completed/void)
- ผู้ใช้ค้นหารายการขายด้วยวันที่หรือหมายเลขออเดอร์
- ผู้ใช้คลิกดูรายละเอียดรายการขาย
- ผู้ใช้ยกเลิกรายการขาย (เฉพาะบิลที่ยังไม่ completed)
- ผู้ใช้ระบุเหตุผลการยกเลิกรายการ
- ผู้ใช้คืนเงิน (เฉพาะบิลที่ยังไม่ completed)
- ระบบอัปเดตสต็อกสินค้าเมื่อยกเลิกหรือคืนเงิน
- ระบบบันทึก void/refund action พร้อมเหตุผลใน Audit Log
- บิลที่มีสถานะ completed ไม่สามารถยกเลิก แก้ไข หรือลบได้
- หากผู้ใช้พยายามยกเลิก/แก้ไข/ลบบิล completed ระบบจะ Block การกระทำ แสดงข้อความแจ้งเตือน บันทึก Audit Log สร้าง Fraud Alert ระดับ critical และส่งแจ้งเตือนผ่าน Telegram (ถ้าตั้งค่าไว้)

### 3.10 การจัดการผู้ใช้

#### 3.10.1 รายการผู้ใช้
- แสดงรายการผู้ใช้ของร้านตัวเองทั้งหมด พร้อมชื่อ ยศที่ได้รับ สถานะใบเตือน (badge สี: เขียว=ปกติ, เหลือง=ใบเหลือง, แดง=ใบแดง/ระงับ)
- Store Owner เพิ่มพนักงานใหม่: กรอกชื่อผู้ใช้, รหัสผ่าน, เลือกยศ (cashier หรือ custom_role)
- Store Owner ไม่สามารถสร้าง store_owner คนอื่นได้
- ผู้ใช้แก้ไขข้อมูลพนักงานในร้านตัวเอง
- ผู้ใช้เปลี่ยนยศของพนักงาน
- ผู้ใช้ลบพนักงาน
- ผู้ใช้คลิกดูประวัติการเตือนของพนักงานแต่ละคน (history dialog)
- ระบบบันทึก user management action ใน Audit Log

#### 3.10.2 ออกใบเตือนพนักงาน
- ผู้ใช้เลือกพนักงานในร้านตัวเองที่ต้องการออกใบเตือน
- ผู้ใช้เลือกประเภทใบเตือน: ใบเหลือง (yellow_card) หรือ ใบแดง (red_card)
- ผู้ใช้กรอกเหตุผลการออกใบเตือน
- ระบบบันทึกใบเตือนใน warning_records: id, user_id, issued_by, warning_type, reason, created_at
- ระบบอัปเดต warning_status ใน profiles ของพนักงาน
- หากออกใบแดง (red_card) ระบบเปลี่ยนสถานะเป็น red_card และระงับการเข้าสู่ระบบ
- ระบบส่งแจ้งเตือนผ่าน Telegram (ถ้าตั้งค่าไว้)
- ระบบบันทึก issue warning action ใน Audit Log

#### 3.10.3 ดูประวัติการเตือน
- ผู้ใช้คลิกดูประวัติการเตือนของพนักงานแต่ละคน
- แสดง dialog รายการใบเตือนทั้งหมด พร้อมวันที่ ประเภทใบเตือน เหตุผล ผู้ออกใบเตือน

### 3.11 การจัดการยศและสิทธิ์ (Role Management)

#### 3.11.1 รายการยศ
- แสดงรายการยศทั้งหมด พร้อมชื่อยศ สีประจำยศ จำนวนผู้ใช้ที่มียศนี้
- แสดงยศ Super Admin และ Store Owner (ลบไม่ได้)
- ผู้ใช้ค้นหายศ

#### 3.11.2 สร้างยศใหม่
- ผู้ใช้กรอกชื่อยศ
- ผู้ใช้เลือกสีประจำยศ
- ผู้ใช้กำหนดสิทธิ์ด้วย checkbox สำหรับแต่ละ permission:
  - view_dashboard (ดู Dashboard)
  - view_reports (ดูรายงาน)
  - view_profit (ดูกำไร)
  - manage_products (จัดการสินค้า)
  - manage_inventory (จัดการสต็อก)
  - manage_users (จัดการผู้ใช้)
  - manage_settings (ตั้งค่าร้าน)
  - process_sales (ขายสินค้า)
  - void_transactions (ยกเลิกรายการ)
  - view_transactions (ดูประวัติการขาย)
  - view_all_transactions (ดูประวัติการขายทั้งหมด)
  - manage_categories (จัดการหมวดหมู่)
  - view_cost (ดูต้นทุน)
  - manage_roles (จัดการยศ)
  - view_audit_log (ดู Audit Log)
  - view_fraud_alerts (ดู Alert การโกง)
  - issue_warnings (ออกใบเตือนพนักงาน)
- ระบบบันทึกยศใหม่
- ระบบบันทึก create role action ใน Audit Log

#### 3.11.3 แก้ไขยศ
- ผู้ใช้เลือกยศที่ต้องการแก้ไข
- ผู้ใช้แก้ไขชื่อยศ สีประจำยศ หรือสิทธิ์
- ระบบบันทึกการเปลี่ยนแปลง
- ระบบบันทึก edit role action ใน Audit Log

#### 3.11.4 ลบยศ
- ผู้ใช้เลือกยศที่ต้องการลบ (ยกเว้นยศ Super Admin และ Store Owner)
- ระบบยืนยันการลบ
- ระบบลบยศออกจากระบบ
- ระบบบันทึก delete role action ใน Audit Log

#### 3.11.5 ยศล่วงหน้า (Role Template)
- แสดงรายการยศล่วงหน้าที่บันทึกไว้ เช่น หัวหน้าแคชเชียร์, ผู้จัดการคลัง, แคชเชียร์ทั่วไป
- ผู้ใช้เลือกยศล่วงหน้าเพื่อนำไปใช้
- ระบบสร้างยศใหม่ตาม template ที่เลือก

### 3.12 ระบบตรวจจับการโกง (Fraud Monitor)

#### 3.12.1 Audit Log
- แสดงรายการ action ของร้านตัวเองทั้งหมด พร้อมวันที่ เวลา ผู้ใช้ action รายละเอียด
- action ที่บันทึก: login, logout, แก้ไขราคา, ยกเลิกรายการ, แก้ไขสต็อก, แก้ไขข้อมูลผู้ใช้, เข้าถึงหน้าต่างๆ, เพิ่ม/แก้ไข/ลบสินค้า, เพิ่ม/แก้ไข/ลบยศ, พยายามแก้ไข/ลบบิล completed, ออกใบเตือนพนักงาน
- ผู้ใช้กรองตาม user, action, วันที่
- ผู้ใช้ค้นหา action

#### 3.12.2 Alert การโกง
- แสดงรายการ alert พฤติกรรมน่าสงสัยของร้านตัวเอง พร้อมระดับความเสี่ยง (Critical/High/Medium/Low)
- แสดง badge ประเภทการโกง เช่น ยักยอกทรัพย์, พฤติกรรมผิดปกติ
- พฤติกรรมน่าสงสัยที่ตรวจจับ:
  - พยายามลบ/แก้ไขบิล completed → Critical (ยักยอกทรัพย์)
  - พยายามแก้ไขสินค้าในบิล completed → Critical (ยักยอกทรัพย์)
  - พยายามแก้ไขราคา/จำนวนใน transaction_items ของบิล completed → Critical (ยักยอกทรัพย์)
  - ยกเลิกรายการมากผิดปกติ (เกิน X ครั้งต่อวัน) → High
  - แก้ไขสต็อกบ่อยโดยไม่มีการขาย → Medium
  - Login ผิดพลาดหลายครั้ง → Medium
  - ขายสินค้าต่ำกว่าราคาขั้นต่ำ (ถ้ามี) → High
  - ลบ/แก้ไขสินค้าหลังการขาย → High
- ผู้ใช้กรองตามระดับความเสี่ยง วันที่ ผู้ใช้ ประเภทการโกง
- ผู้ใช้คลิกดูรายละเอียด alert

#### 3.12.3 สถิติพฤติกรรมน่าสงสัย
- แสดงสถิติน่าสงสัยต่อพนักงานแต่ละคนในร้านตัวเอง
- แสดงจำนวนการยกเลิกรายการ จำนวนการแก้ไขสต็อก จำนวน login ผิดพลาด จำนวนครั้งที่พยายามแก้ไข/ลบบิล completed
- แสดงประวัติการแก้ไขสต็อกทั้งหมด พร้อมเหตุผล
- ผู้ใช้เลือกช่วงเวลา
- ผู้ใช้เลือกพนักงาน

### 3.13 การตั้งค่าร้าน

#### 3.13.1 ข้อมูลร้าน
- ผู้ใช้กรอกชื่อร้าน
- ผู้ใช้กรอกที่อยู่ร้าน
- ผู้ใช้กรอกเบอร์โทรศัพท์

#### 3.13.2 การตั้งค่าทั่วไป

- ผู้ใช้เปิด/ปิดการพิมพ์ใบเสร็จอัตโนมัติ
- ผู้ใช้ตั้งค่าจำนวนสต็อกขั้นต่ำสำหรับแจ้งเตือน
- ผู้ใช้ตั้งค่าจำนวนครั้งยกเลิกรายการสูงสุดต่อวัน (สำหรับ Fraud Detection)

- ระบบทดสอบการเชื่อมต่อ Telegram Bot
- ระบบแสดงสถานะการเชื่อมต่อ

### 3.14 Telegram Bot Integration

#### 3.14.1 การแจ้งเตือนอัตโนมัติ
- เมื่อตรวจพบ Fraud Alert ใหม่ (ทุก severity) ระบบส่งแจ้งเตือนผ่าน Telegram
- เมื่อพนักงานพยายามยักยอกทรัพย์ (transaction_tampering) ระบบส่งแจ้งเตือนด่วนผ่าน Telegram
- เมื่อออกใบเตือนพนักงาน ระบบส่งแจ้งเตือนผ่าน Telegram
- ระบบใช้ Edge Function เรียก Telegram Bot API ส่ง message ไปยัง Chat ID ที่กำหนด

#### 3.14.2 การสอบถามข้อมูลผ่าน Telegram Bot

  - ยอดขายวันนี้
  - ยอดขายสัปดาห์นี้
  - ยอดขายช่วงวันที่ที่กำหนด
- ระบบตอบกลับข้อมูลผ่าน Telegram

- ระบบแปลทุก label, ข้อความ, toast, error message ตามภาษาที่เลือก
- ระบบบันทึกภาษาที่เลือกใน localStorage หรือ user preferences

## 4. กฎทางธุรกิจและตรรกะ

### 4.1 สถาปัตยกรรม Multi-tenant
- ระบบรองรับหลายร้านค้า (tenant) บนเซิร์ฟเวอร์เดียว
- แต่ละร้านค้ามี store_id เฉพาะ
- ทุก table ที่เกี่ยวข้องกับข้อมูลร้าน (products, transactions, users, inventory, audit_log, fraud_alerts) ต้องมี store_id
- Row Level Security (RLS) ต้องกรองด้วย store_id ทุก table
- Store Owner และพนักงานเห็นเฉพาะข้อมูลของร้านตัวเอง (store_id ตรงกัน)
- Super Admin เห็นข้อมูลทุกร้าน (ไม่กรอง store_id)

### 4.2 ลำดับชั้นผู้ใช้
- super_admin > store_owner > custom_roles > cashier
- Super Admin: มีสิทธิ์ทุกอย่างข้ามร้าน สร้าง/เปิด/ปิดร้าน สร้าง account store_owner ลบไม่ได้
- Store Owner: มีสิทธิ์ทุกอย่างในร้านตนเอง เพิ่ม/จัดการพนักงาน ไม่สามารถสร้าง store_owner คนอื่นได้ ลบไม่ได้
- Cashier / Custom Role: สิทธิ์ตามที่กำหนดใน Role Management
- ผู้ใช้แต่ละคนมีได้ 1 ยศ
- ระบบตรวจสอบสิทธิ์ก่อนอนุญาตให้เข้าถึงฟังก์ชัน

### 4.3 การคำนวณราคา
- ราคารวม = ราคาสินค้า × จำนวน
- ยอดรวมทั้งหมด = ผลรวมของราคารวมทุกรายการ
- ภาษี VAT = ยอดรวมทั้งหมด × เปอร์เซ็นต์ VAT
- ยอดชำระ = ยอดรวมทั้งหมด + ภาษี VAT

### 4.4 การจัดการสต็อก
- เมื่อขายสินค้าสำเร็จ ระบบลดสต็อกสินค้าตามจำนวนที่ขาย
- เมื่อยกเลิกรายการขายหรือคืนเงิน ระบบเพิ่มสต็อกสินค้ากลับ
- เมื่อสต็อกสินค้าต่ำกว่าจำนวนขั้นต่ำที่กำหนด ระบบแจ้งเตือน
- การแก้ไขสต็อกต้องระบุเหตุผลและบันทึกใน Audit Log

### 4.5 การคำนวณกำไร
- กำไรต่อรายการ = (ราคาขาย - ต้นทุน) × จำนวน
- กำไรรวม = ผลรวมของกำไรต่อรายการทุกรายการในช่วงเวลาที่เลือก
- อัตรากำไร = (กำไรรวม / ยอดขายรวม) × 100

### 4.6 การบันทึกรายการขาย
- ระบบบันทึกข้อมูล: วันที่, เวลา, รายการสินค้า, จำนวน, ราคา, ยอดรวม, วิธีชำระเงิน, ผู้ขาย, สถานะ (completed/void), store_id
- ระบบสร้างหมายเลขออเดอร์อัตโนมัติ
- เมื่อชำระเงินสำเร็จหรือพิมพ์ใบเสร็จ ระบบเปลี่ยนสถานะเป็น completed
- ระบบบันทึก action ใน Audit Log

### 4.7 การป้องกันการโกง
- ราคาสินค้าในตะกร้ามาจากฐานข้อมูลเท่านั้น ไม่สามารถแก้ไขได้
- การยกเลิกรายการต้องระบุเหตุผล
- การแก้ไขสต็อกต้องระบุเหตุผล
- ระบบบันทึกทุก action ที่สำคัญใน Audit Log
- ระบบตรวจจับพฤติกรรมน่าสงสัยและสร้าง alert อัตโนมัติ

### 4.8 การตรวจจับพฤติกรรมน่าสงสัย
- ยกเลิกรายการเกินจำนวนที่กำหนดต่อวัน → สร้าง alert ระดับ High
- แก้ไขสต็อกบ่อยโดยไม่มีการขาย → สร้าง alert ระดับ Medium
- Login ผิดพลาดหลายครั้ง → สร้าง alert ระดับ Medium
- ขายสินค้าต่ำกว่าราคาขั้นต่ำ → สร้าง alert ระดับ High
- ลบ/แก้ไขสินค้าหลังการขาย → สร้าง alert ระดับ High

### 4.9 ระบบป้องกันการยักยอกทรัพย์ (Transaction Integrity)
- บิลที่มีสถานะ completed ถือว่าปิดแล้ว ไม่สามารถแก้ไข ลบ หรือยกเลิกได้
- หากผู้ใช้พยายามลบ/แก้ไขบิล completed:
  - ระบบ Block การกระทำทันที
  - ระบบแสดงข้อความแจ้งเตือน
  - ระบบบันทึก action ใน Audit Log
  - ระบบสร้าง Fraud Alert ระดับ Critical พร้อม badge ยักยอกทรัพย์
  - ระบบส่งแจ้งเตือนผ่าน Telegram (ถ้าตั้งค่าไว้)
- หากผู้ใช้พยายามแก้ไขสินค้าในบิล completed:
  - ระบบ Block การกระทำทันที
  - ระบบแสดงข้อความแจ้งเตือน
  - ระบบบันทึก action ใน Audit Log
  - ระบบสร้าง Fraud Alert ระดับ Critical พร้อม badge ยักยอกทรัพย์
  - ระบบส่งแจ้งเตือนผ่าน Telegram (ถ้าตั้งค่าไว้)
- หากผู้ใช้พยายามแก้ไขราคา/จำนวนใน transaction_items ของบิล completed:
  - ระบบ Block การกระทำทันที
  - ระบบแสดงข้อความแจ้งเตือน
  - ระบบบันทึก action ใน Audit Log
  - ระบบสร้าง Fraud Alert ระดับ Critical พร้อม badge ยักยอกทรัพย์
  - ระบบส่งแจ้งเตือนผ่าน Telegram (ถ้าตั้งค่าไว้)
- ระบบใช้ Row Level Security ในฐานข้อมูล: transaction_items ของ completed transactions ห้าม UPDATE/DELETE จาก client

### 4.10 การจัดการรูปภาพสินค้า
- รูปภาพสินค้าเป็น optional ไม่บังคับใส่
- รองรับการอัปโหลดจาก Gallery และถ่ายรูปจากกล้องโดยตรง
- รูปภาพอัปโหลดไปยัง Supabase Storage bucket product-images
- จัดเก็บในโฟลเดอร์ products/{product_id}/{filename}
- ระบบบันทึก image_url เป็น public URL จาก Supabase Storage ใน products table
- แสดงรูปภาพใน ProductsPage (thumbnail ในตาราง) และ POSPage (รูปสินค้าในกริด) จาก public URL

### 4.11 ระบบใบเตือนพนักงาน (Warning System)
- พนักงานแต่ละคนมีสถานะใบเตือน: normal (ปกติ), yellow_card (ใบเหลือง), red_card (ใบแดง/ระงับ)
- Store Owner สามารถออกใบเตือนให้พนักงาน: yellow_card หรือ red_card
- ระบุเหตุผลการออกใบเตือนได้
- ใบแดง (red_card) = ระงับการเข้าสู่ระบบ (ล็อคบัญชี)
- ระบบบันทึกใบเตือนใน warning_records: id, user_id, issued_by, warning_type, reason, created_at
- ระบบอัปเดต warning_status ใน profiles
- ระบบแสดง badge สีตามสถานะในหน้า UsersPage: เขียว=normal, เหลือง=yellow_card, แดง=red_card
- ผู้ใช้สามารถดูประวัติการเตือนของพนักงานแต่ละคน (history dialog)
- เมื่อพนักงานมีสถานะ red_card ระบบ block การ login และแสดงข้อความ (แปลตามภาษาที่เลือก)

### 4.12 Telegram Bot Integration
- Store Owner ตั้งค่า Telegram Bot Token และ Chat ID ในหน้า Settings
- ระบบบันทึก telegram_bot_token และ telegram_chat_id ใน shop_settings
- ระบบส่งแจ้งเตือนผ่าน Telegram อัตโนมัติในกรณี:
  - ตรวจพบ Fraud Alert ใหม่ (ทุก severity)
  - พนักงานพยายามยักยอกทรัพย์ (transaction_tampering) → แจ้งเตือนด่วน
  - ออกใบเตือนพนักงาน
- Store Owner สามารถสอบถามข้อมูลผ่าน Telegram Bot:
  - ยอดขายวันนี้
  - ยอดขายสัปดาห์นี้
  - ยอดขายช่วงวันที่ที่กำหนด
- ระบบใช้ Edge Function เรียก Telegram Bot API ส่ง message ไปยัง Chat ID
- Super Admin สามารถดูและแก้ไข Telegram Token ของทุกร้านในหน้า Super Admin Panel

### 4.13 ระบบหลายภาษา (i18n)
- ระบบรองรับ 4 ภาษา: ไทย (th), ลาว (lo), อังกฤษ (en), จีน (zh)
- ใช้ i18next หรือ react-i18next สำหรับ translation
- แสดงปุ่มธงชาติที่ header/navbar ทุกหน้า รวมถึงหน้า login
- ผู้ใช้คลิกธงชาติเพื่อสลับภาษา
- ระบบแปลทุก label, ข้อความ, toast, error message ตามภาษาที่เลือก
- ระบบบันทึกภาษาที่เลือกใน localStorage หรือ user preferences
- ภาษาเริ่มต้น: ภาษาไทย

| พยายามแก้ไขสินค้าในบิล completed | ระบบ Block การกระทำ แสดงข้อความแจ้งเตือน บันทึก Audit Log สร้าง Fraud Alert ระดับ Critical ส่งแจ้งเตือนผ่าน Telegram |
| พยายามแก้ไขราคา/จำนวนใน transaction_items ของบิล completed | ระบบ Block การกระทำ แสดงข้อความแจ้งเตือน บันทึก Audit Log สร้าง Fraud Alert ระดับ Critical ส่งแจ้งเตือนผ่าน Telegram |

## 6. เกณฑ์การตรวจรับ

1. Super Admin เข้าสู่ระบบและเข้าหน้า Super Admin Panel
2. Super Admin สร้างร้านใหม่พร้อมสร้าง account store_owner
3. Super Admin ดูรายการร้านค้าทั้งหมดและสถานะ (active/inactive)
4. Super Admin เปิด/ปิดร้านใดร้านหนึ่ง
5. Super Admin ดู Audit Log และ Fraud Alert ของทุกร้าน
6. Super Admin จัดการ Telegram Token ของร้านใดร้านหนึ่ง
7. Store Owner เข้าสู่ระบบและเข้าหน้า Dashboard เห็นเฉพาะข้อมูลร้านตัวเอง
8. Store Owner เพิ่มพนักงานใหม่ (cashier) ในร้านตัวเอง
9. Store Owner พยายามสร้าง account store_owner คนอื่น ระบบ block
10. Store Owner เข้าหน้าการจัดการผู้ใช้ เห็น badge สีแสดงสถานะใบเตือนของพนักงาน
11. Store Owner ออกใบเหลือง (yellow_card) ให้พนักงานคนหนึ่ง พร้อมระบุเหตุผล
12. Store Owner ดูประวัติการเตือนของพนักงานคนนั้น เห็นรายการใบเตือนที่ออกไป
13. Store Owner ออกใบแดง (red_card) ให้พนักงานอีกคนหนึ่ง
14. พนักงานที่ได้รับใบแดงพยายาม login ระบบ block และแสดงข้อความ (แปลตามภาษาที่เลือก)
15. Store Owner เข้าหน้าการตั้งค่าร้าน ตั้งค่า Telegram Bot Token และ Chat ID
16. ระบบทดสอบการเชื่อมต่อ Telegram Bot สำเร็จ
17. พนักงานพยายามแก้ไขบิล completed ระบบ block การกระทำ สร้าง Fraud Alert ระดับ Critical และส่งแจ้งเตือนผ่าน Telegram
18. Store Owner เปิด Telegram เห็นข้อความแจ้งเตือนการพยายามยักยอกทรัพย์
19. Store Owner ส่งข้อความถามยอดขายวันนี้ผ่าน Telegram Bot ระบบตอบกลับยอดขายวันนี้
20. Store Owner เข้าหน้าการจัดการสินค้า เพิ่มสินค้าใหม่พร้อมอัปโหลดรูปภาพจากกล้อง
21. Store Owner เข้าหน้าขายสินค้า เห็นรูปภาพสินค้าที่อัปโหลดแสดงในกริดสินค้าถูกต้อง
22. พนักงานขายเข้าสู่ระบบด้วยบทบาท Cashier เห็นเฉพาะสินค้าของร้านตัวเอง
23. พนักงานขายเลือกสินค้าหรือสแกนบาร์โค้ดเพื่อเพิ่มสินค้าเข้าตะกร้า
24. พนักงานขายเลือกวิธีชำระเงิน (เงินสด/บัตร/QR Code) และชำระเงิน
25. ระบบบันทึกรายการขาย อัปเดตสต็อก เปลี่ยนสถานะเป็น completed และแสดงหน้าจอยืนยันการขายสำเร็จ
26. ผู้ใช้คลิกปุ่มธงชาติที่ header เพื่อสลับภาษาเป็นภาษาอังกฤษ
27. ระบบแปลทุก label, ข้อความ, toast, error message เป็นภาษาอังกฤษ
28. ผู้ใช้สลับภาษากลับเป็นภาษาไทย ระบบแปลกลับเป็นภาษาไทย

## 7. ฟังก์ชันที่ไม่ดำเนินการในเวอร์ชันนี้

- การเชื่อมต่อกับเครื่องพิมพ์ใบเสร็จแบบฮาร์ดแวร์
- การซิงค์ข้อมูลระหว่างหลายสาขา
- การจัดการโปรโมชั่นและส่วนลด
- การจัดการสมาชิกและโปรแกรมสะสมแต้ม
- การเชื่อมต่อกับระบบบัญชี
- การสั่งซื้อสินค้าจากซัพพลายเออร์
- การจัดการพนักงานและตารางเวลาทำงาน
- การรองรับหลายสกุลเงิน
- การวิเคราะห์ข้อมูลขั้นสูง (AI/ML)
- การแจ้งเตือนผ่าน SMS หรือ Email
- การสำรองข้อมูลอัตโนมัติ
- โหมดออฟไลน์
- การกำหนดยศหลายยศต่อผู้ใช้
- การตั้งค่า permission แบบละเอียดต่อผู้ใช้ (นอกเหนือจากยศ)
- การส่งออกรายงาน Audit Log
- การตั้งค่า alert แบบกำหนดเอง
- การแก้ไขรูปภาพสินค้าภายในระบบ (crop/resize)
- การจัดการรูปภาพหลายรูปต่อสินค้า
- การยกเลิกใบเตือนพนักงาน (ใบเหลือง/ใบแดง)
- การตั้งค่าระดับใบเตือนแบบกำหนดเอง
- การส่งแจ้งเตือนผ่าน Telegram แบบกำหนดเอง
- การแปลภาษาอัตโนมัติด้วย AI
- การรองรับภาษาอื่นนอกเหนือจาก 4 ภาษาที่กำหนด
